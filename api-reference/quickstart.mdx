## Overview

This guide walks you through creating a complete ledger system, from initial setup to recording your first transaction. By the end, you'll understand the basic workflow and be ready to integrate Transfa into your application.

## Prerequisites

<Steps>
  <Step title="API Access">
    Ensure you have access to a Transfa instance (local or hosted)
  </Step>
  <Step title="Authentication">
    Obtain an API key or admin credentials
  </Step>
  <Step title="HTTP Client">
    Have curl, Postman, or similar HTTP client ready
  </Step>
</Steps>

## Step 1: Create a Ledger

A ledger is the top-level container for your accounts and transactions. Start by creating one:

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/v1/ledgers \
  -H "Content-Type: application/json" \
  -u key_id:key_secret \
  -d '{
    "name": "Main Ledger",
    "description": "Primary accounting ledger",
    "metadata": {
      "environment": "production"
    }
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/v1/ledgers', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Basic ' + btoa('key_id:key_secret')
  },
  body: JSON.stringify({
    name: 'Main Ledger',
    description: 'Primary accounting ledger',
    metadata: {
      environment: 'production'
    }
  })
});

const ledger = await response.json();
console.log('Ledger ID:', ledger.id);
```

```python Python
import requests
from requests.auth import HTTPBasicAuth

response = requests.post(
    'http://localhost:3000/v1/ledgers',
    auth=HTTPBasicAuth('key_id', 'key_secret'),
    json={
        'name': 'Main Ledger',
        'description': 'Primary accounting ledger',
        'metadata': {
            'environment': 'production'
        }
    }
)

ledger = response.json()
print(f"Ledger ID: {ledger['id']}")
```
</CodeGroup>

**Response:**
```json
{
  "id": "01933eb1-5fc4-7c85-9c7e-12a34b56c789",
  "name": "Main Ledger",
  "description": "Primary accounting ledger",
  "metadata": {
    "environment": "production"
  },
  "createdAt": "2025-10-20T10:00:00Z",
  "updatedAt": "2025-10-20T10:00:00Z"
}
```

<Note>
Save the ledger `id` - you'll need it for creating accounts and transactions.
</Note>

## Step 2: Create a Currency

Define the currency your accounts will use:

```bash
curl -X POST http://localhost:3000/v1/currencies \
  -H "Content-Type: application/json" \
  -u key_id:key_secret \
  -d '{
    "code": "USD",
    "name": "United States Dollar",
    "scale": 2
  }'
```

**Response:**
```json
{
  "code": "USD",
  "name": "United States Dollar",
  "scale": 2
}
```

<Info>
If the currency already exists, you'll receive a 409 Conflict error. That's okay - you can proceed to the next step.
</Info>

## Step 3: Create Accounts

Create two accounts to demonstrate a simple payment flow:

### Cash Account (Asset - Debit)

```bash
curl -X POST http://localhost:3000/v1/ledger_accounts \
  -H "Content-Type: application/json" \
  -u key_id:key_secret \

    "externalId": "cash_account",
    "metadata": {
      "type": "asset",
```

## Step 4: Record a Transaction

Now record a transaction representing a customer payment of $500.00:

```bash
curl -X POST http://localhost:3000/v1/ledger_transactions \
  -H "Content-Type: application/json" \
  -H "idempotency-key: payment-001" \
  -u key_id:key_secret \
  -d '{
    "ledgerId": "01933eb1-5fc4-7c85-9c7e-12a34b56c789",
    "description": "Customer payment received",
    "externalId": "payment_001",
    "metadata": {
      "customerId": "cust_789",
      "invoiceId": "inv_456",
      "paymentMethod": "bank_transfer"
    },
    "entries": [
      {
        "accountId": "01933eb2-a1c5-7d42-8f3a-9b87c6d5e4f3",
        "amount": 500.00,
        "direction": "debit",
        "description": "Cash received"
      },
      {
        "accountId": "01933eb2-b7d8-7f53-9a2e-7c8d9e0f1a2b",
        "amount": 500.00,
        "direction": "credit",
        "description": "Revenue recognized"
      }
    ]
  }'
```

**Response:**
```json
{
  "id": "01933eb3-c2f4-7a91-b5c8-3d4e5f6a7b8c",
  "ledgerId": "01933eb1-5fc4-7c85-9c7e-12a34b56c789",
  "status": "POSTED",
  "postedAt": "2025-10-20T10:05:00Z",
  "description": "Customer payment received",
  "externalId": "payment_001",
  "metadata": {
    "customerId": "cust_789",
    "invoiceId": "inv_456",
    "paymentMethod": "bank_transfer"
  },
  "entries": [
    {
      "id": "01933eb3-d5a7-7b82-a9f4-8e9f0a1b2c3d",
      "accountId": "01933eb2-a1c5-7d42-8f3a-9b87c6d5e4f3",
      "amount": 500.00,
      "direction": "debit",
      "description": "Cash received"
    },
    {
      "id": "01933eb3-e8c9-7d73-b2e5-9f0a1b2c3d4e",
      "accountId": "01933eb2-b7d8-7f53-9a2e-7c8d9e0f1a2b",
      "amount": 500.00,
      "direction": "credit",
      "description": "Revenue recognized"
    }
  ],
  "createdAt": "2025-10-20T10:05:00Z"
}
```

<Info>
Transactions are **posted by default** and immediately update account balances. You can optionally create a transaction with `"status": "PENDING"` if you need to review it before posting.
</Info>

## Step 5: Verify Account Balances

Check that the account balances have been updated:

```bash
# Check Cash account
curl http://localhost:3000/v1/ledger_accounts/01933eb2-a1c5-7d42-8f3a-9b87c6d5e4f3 \
  -u key_id:key_secret

# Check Revenue account
curl http://localhost:3000/v1/ledger_accounts/01933eb2-b7d8-7f53-9a2e-7c8d9e0f1a2b \
  -u key_id:key_secret
```
